3//! Native helpers for disk-based VLA export archives.
4//!
5//! This crate starts the native Rust implementation of the CASA
6//! `importvla` workflow. The current wave is intentionally limited to
7//! disk-based archive files such as `*.xp1`/`*.xp5`, matching the user's
8//! current scope and avoiding the legacy online/tape branches present in
9//! CASA's C++ `VLAFiller`.
10//!
11//! The crate currently provides:
12//!
13//! - task-style option types mirroring CASA `importvla`
14//! - logical-record reassembly for VLA disk export files
15//! - Record Control Area (RCA) decoding
16//! - archive scan summaries suitable for later importer planning/parity work
17
