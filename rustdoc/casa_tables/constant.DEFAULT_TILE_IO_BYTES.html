Default physical I/O target for newly created tiled arrays and columns.

Four MiB is large enough to sustain sequential local-disk throughput while keeping a small fixed number of tiles comfortably bounded in memory. The value controls new-file layout only; readers always honor the tile shape persisted in an existing casacore table.
