2//! Astronomical measures: typed physical quantities with reference frames.
3//!
4//! This module provides the casacore *measures* system — physical quantities
5//! (epochs, positions, directions, frequencies, Doppler shifts) that carry
6//! both a numeric value and a reference frame. Reference-frame conversions
7//! use the IAU SOFA algorithms via the [`sofars`] crate.
8//!
9//! ## Key types
10//!
11//! - [`MEpoch`] — a time instant in one of 12 time scales (UTC, TAI, TT, etc.),
12//!   equivalent to C++ `casa::MEpoch`.
13//! - [`MPosition`] — a 3D position in ITRF or WGS84, equivalent to C++
14//!   `casa::MPosition`.
15//! - [`MDirection`](direction::MDirection) — a sky direction in J2000, GALACTIC,
16//!   etc., equivalent to C++ `casa::MDirection`.
17//! - [`MFrequency`](frequency::MFrequency) — a spectral frequency in LSRK,
18//!   BARY, TOPO, etc., equivalent to C++ `casa::MFrequency`.
19//! - [`MDoppler`](doppler::MDoppler) — a Doppler shift in RADIO, Z, BETA, etc.,
20//!   equivalent to C++ `casa::MDoppler`.
21//! - [`MeasFrame`] — conversion context carrying epoch, position, direction,
22//!   and UT1−UTC offset.
23//! - [`MjdHighPrec`] — high-precision Modified Julian Date (day + fraction),
24//!   equivalent to C++ `MVEpoch`.
25//!
26//! ## Record serialization
27//!
28//! The [`record`] submodule provides `to_record` / `from_record` functions
29//! that produce [`RecordValue`](crate::RecordValue) in the same layout as C++
30//! `MeasureHolder`, ensuring on-disk compatibility.
31//!
32//! ## Examples
33//!
34//! ```
35//! use casacore_types::measures::{MEpoch, EpochRef, MeasFrame};
36//!
37//! // Convert J2000.0 from UTC to TAI
38//! let utc = MEpoch::from_mjd(51544.5, EpochRef::UTC);
39//! let frame = MeasFrame::new();
40//! let tai = utc.convert_to(EpochRef::TAI, &frame).unwrap();
41//! let diff_s = (tai.value().as_mjd() - utc.value().as_mjd()) * 86400.0;
42//! assert!((diff_s - 32.0).abs() < 0.01);
43//! ```
