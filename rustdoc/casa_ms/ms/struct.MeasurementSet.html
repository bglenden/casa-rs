Struct MeasurementSet 

Source 
pub struct MeasurementSet { /* private fields */ }
Expand description

A MeasurementSet: main table + subtables.

This is the top-level type for working with MS data. It owns the main table and all attached subtables (12 required + 5 optional).

When saved to disk, subtable links are emitted as true casacore TpTable keywords, using the same relative-path encoding that C++ MeasurementSet writes for sibling subtables.

§Example

use casa_ms::ms::MeasurementSet;
use casa_ms::builder::MeasurementSetBuilder;

// Create an in-memory MS
let ms = MeasurementSet::create_memory(MeasurementSetBuilder::new())
    .expect("create MS");
assert_eq!(ms.row_count(), 0);
assert_eq!(ms.subtable_ids().len(), 12);

pub fn save(&mut self) -> MsResult<()>

Save the MS to its path (must have been created with a path or opened from disk).

The save step refreshes the main-table metadata and rewrites subtable keyword payloads using casacore’s relative ././SUBTABLE form.

Source

pub fn save_assuming_valid(&mut self) -> MsResult<()>

Save the MS to its path without re-validating each table against its schema.

This preserves the same on-disk layout as save, but skips the extra full-table validation pass for the main table and every subtable. It is intended for advanced writers that already route all mutations through schema-aware code paths and want to avoid rechecking every row during the final serialization step.

Source

pub fn save_main_table_only(&mut self) -> MsResult<()>

Save only the MS main table back to disk.

This is intended for workflows that mutate only MAIN columns/keywords while leaving all subtables untouched, such as the initial applycal-class calibration executor. The method still refreshes main-table metadata and subtable-link keywords before writing the main table, but it skips rewriting the individual subtable directories.

Source

pub fn save_main_table_only_assuming_valid(&mut self) -> MsResult<()>

Save only the MS main table back to disk, assuming MAIN is already schema-valid.

This preserves the same on-disk layout as save_main_table_only but skips the extra full-table validation pass before serializing the main table. It is intended for write paths that mutate MAIN only through validating APIs and want to avoid rechecking every row.

Source

pub fn save_as(&mut self, path: impl AsRef<Path>) -> MsResult<()>

Save the MS to a new path.

All persisted subtable references are rebased so the copied MS remains self-consistent when reopened by Rust or C++ casacore tools.
