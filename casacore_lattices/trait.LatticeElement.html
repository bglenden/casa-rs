Expand description

Marker trait for types that can be stored as elements of a Lattice.

This corresponds to the template constraint on C++ Lattice<T>: only the 12 casacore-native types (bool, u8, i16, u16, i32, u32, i64, f32, f64, Complex, DComplex, String) are valid lattice elements.

Each implementing type provides its [PrimitiveType] tag and a default (zero/empty) value for initializing lattice storage.

§Examples

use casacore_lattices::LatticeElement;
use casacore_types::PrimitiveType;

assert_eq!(f64::PRIMITIVE_TYPE, PrimitiveType::Float64);
assert_eq!(f64::default_value(), 0.0);

Required Methods§

Source

fn default_value() -> Self

Returns the default (zero/empty) value for this element type.

Used when initializing lattice storage. Mirrors C++ casacore’s default-initialization for Array<T>.

Dyn Compatibility§

This trait is not dyn compatible.

In older versions of Rust, dyn compatibility was called "object safety", so this trait is not object safe.
