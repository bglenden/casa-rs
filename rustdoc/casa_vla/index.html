Native helpers for disk-based VLA export archives.

This crate starts the native Rust implementation of the CASA importvla workflow. The current wave is intentionally limited to disk-based archive files such as *.xp1/*.xp5, matching the user’s current scope and avoiding the legacy online/tape branches present in CASA’s C++ VLAFiller.

The crate currently provides:

  • task-style option types mirroring CASA importvla
  • logical-record reassembly for VLA disk export files
  • Record Control Area (RCA) decoding
  • archive scan summaries suitable for later importer planning/parity work
