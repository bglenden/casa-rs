2//! Calibration-table compatibility support for `casa-rs`.
3//!
4//! This crate starts the calibration workflow substrate planned for
5//! `casars`. The first wave focuses on reading and validating CASA-produced
6//! calibration tables, normalising the metadata needed for later apply/solve
7//! work, and emitting machine-readable summaries that can anchor slow parity
8//! tests against CASA.
9//!
10//! The public API is intentionally narrow in the first wave:
11//!
12//! - [`summarize_table`] opens a calibration table written by CASA or Rust and
13//!   returns a normalized [`CalibrationTableSummary`].
14//! - [`summarize_tables`] batches the same operation for CLI callers.
15//! - [`CalibrationTableSummary::supported_for_v1_apply`] indicates whether the
16//!   table falls inside the initial `Complex`/`CPARAM` family targeted by the
17//!   upcoming `applycal`-class workflow.
18//!
19//! The implementation is built on [`casa_tables::Table`] rather than a new
20//! storage stack, so every on-disk assumption made here is exercised against
21//! the same table reader/writer substrate used elsewhere in the repo.
