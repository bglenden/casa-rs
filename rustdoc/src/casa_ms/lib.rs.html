3//! Typed MeasurementSet (MS) API for radio interferometry data.
4//!
5//! This crate provides idiomatic Rust APIs for reading and writing
6//! MeasurementSet files — the primary data format for radio interferometry.
7//! It builds on [`casa_tables`] for table I/O and [`casa_types`] for
8//! value types, measures, and quanta.
9//!
10//! The on-disk layout is intended to remain interoperable with C++ casacore:
11//! subtable links are emitted as real `TpTable` keywords using casacore's
12//! relative-path conventions, and required unit / measure metadata is attached
13//! to the standard MS columns.
14//!
15//! # Architecture
16//!
17//! The C++ MeasurementSet uses template inheritance with enum metadata maps.
18//! This Rust design replaces that with:
19//!
20//! - **`const` arrays of [`ColumnDef`](column_def::ColumnDef)** for schema metadata
21//! - **[`SubtableId`]** enum for the 17 subtables
22//! - **Typed wrapper structs** for the standard subtables, holding `&Table`
23//!   or `&mut Table` references
24//!
25//! # Schema modules
26//!
27//! The [`schema`] module and its submodules define all column and keyword
28//! metadata as compile-time constants. Each subtable module exports
29//! `REQUIRED_COLUMNS` and `OPTIONAL_COLUMNS` arrays that exactly match
30//! the C++ `MS*Enums.h` headers.
31//!
32//! # Example
33//!
34//! ```rust
35//! use casa_ms::schema::{SubtableId, required_columns};
36//! use casa_ms::column_def::build_table_schema;
37//!
38//! // Build a TableSchema for the ANTENNA subtable
39//! let schema = build_table_schema(required_columns(SubtableId::Antenna))
40//!     .expect("valid schema");
41//! assert_eq!(schema.columns().len(), 8);
42//! ```
