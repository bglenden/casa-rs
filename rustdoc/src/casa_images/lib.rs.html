2//! casacore-compatible typed images and related metadata.
3//!
4//! # Overview
5//!
6//! This crate provides the Rust equivalents of the C++ casacore image classes:
7//!
8//! | Rust type                | C++ equivalent          | Notes                              |
9//! |--------------------------|-------------------------|------------------------------------|
10//! | [`PagedImage<T>`]        | `PagedImage<T>`         | Persistent, table-backed image     |
11//! | [`TempImage<T>`]         | `TempImage<T>`          | In-memory or scratch-disk image    |
12//! | [`ImageExpr<T>`]         | `ImageExpr<T>`          | Lazy read-only expression image    |
13//! | [`SubImage`]/[`SubImageMut`] | `SubImage<T>`       | View-style image sections          |
14//! | [`MaskExpr<T>`]          | `LatticeExprNode`       | Lazy boolean expression            |
15//! | [`ImageInfo`]            | `ImageInfo`             | Image metadata (beam, type, etc.)  |
16//! | [`ImageBeamSet`]         | `ImageBeamSet`          | Beam descriptions                  |
17//!
18//! - [`PagedImage<T>`] stores pixel data in a casacore `Table` with
19//!   `TiledCellStMan`. Suitable for persistent images on disk.
20//! - [`TempImage<T>`] stores pixels in a [`TempLattice`](casa_lattices::TempLattice)
21//!   and metadata in member fields — no `Table` involved. Small images stay
22//!   in memory; larger ones spill to scratch disk. Convert to `PagedImage`
23//!   via [`save_as()`](TempImage::save_as) or [`into_paged()`](TempImage::into_paged).
24//! - [`ImageExpr<T>`] evaluates lazily; pixels are computed only for the
25//!   requested point or region.
26//! - [`SubImage`] and [`SubImageMut`] are view-style image sections.
27//! - Current parity boundary: full `CoordinateSystem` persistence and
28//!   cross-language reconstruction are still incomplete. Pixel data, units,
29//!   masks, and [`ImageInfo`] interoperate more reliably today than
30//!   coordinate metadata.
31//!
32//! Supported image pixel types: [`f32`], [`f64`],
33//! [`Complex32`](casa_types::Complex32), and
34//! [`Complex64`](casa_types::Complex64). The convenience alias [`Image`]
35//! remains available for `PagedImage<f32>`.
36//!
37//! # Example
38//!
39//! ```rust
40//! use casa_coordinates::CoordinateSystem;
41//! use casa_images::{
42//!     ImageExpr, ImageExprBinaryOp, ImageExprUnaryOp, ImageIter, ImageType, TempImage,
43//! };
44//! use casa_lattices::{Lattice, LatticeMut};
45//! use casa_types::Complex32;
46//!
47//! let mut image = TempImage::<f32>::new(vec![8, 8], CoordinateSystem::new()).unwrap();
48//! image.set(1.0).unwrap();
49//!
50//! let expr = ImageExpr::from_image(&image)
51//!     .unwrap()
52//!     .binary_scalar(2.0, ImageExprBinaryOp::Multiply)
53//!     .unary(ImageExprUnaryOp::Negate);
54//! assert_eq!(expr.get_at(&[0, 0]).unwrap(), -2.0);
55//!
56//! let complex = TempImage::<Complex32>::new(vec![4, 4], CoordinateSystem::new()).unwrap();
57//! let _chunks = ImageIter::new(&complex, vec![2, 2]).count();
58//! assert_eq!(ImageType::Velocity.to_string(), "Velocity");
59//! ```
