Atomically write an explicitly named sparse profile.

Unlike managed Last storage this does not take a per-surface lock: the destination is user-selected and a failed concurrent Save As remains an explicit error. The temporary file is nevertheless created beside the destination, flushed, and atomically renamed so readers never observe a partial profile.
