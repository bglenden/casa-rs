2//! N-dimensional lattice abstractions for casacore-compatible data.
3//!
4//! This crate provides the `Lattice<T>` and `LatticeMut<T>` traits for
5//! typed, N-dimensional array access, along with concrete implementations:
6//!
7//! - [`ArrayLattice<T>`] — in-memory lattice wrapping an `ArrayD<T>`.
8//! - [`PagedArray<T>`] — disk-backed lattice using tiled table storage.
9//! - [`TempLattice<T>`] — automatic memory/disk switching lattice.
10//! - [`LatticeStatistics<T>`] — cached statistics over lattices, with explicit
11//!   Rust execution-policy control via [`ExecutionPolicy`].
12//!
13//! # Iteration
14//!
15//! Navigators ([`LatticeStepper`], [`TiledLineStepper`], [`TileStepper`])
16//! define low-level stepping strategies. [`TraversalSpec`] provides a
17//! higher-level traversal API, while [`LatticeIter`] and [`LatticeIterMut`]
18//! remain available for explicit navigator-based iteration. The
19//! [`LatticeIterExt`] trait provides convenience methods
20//! (`traverse`, `iter_lines`, `iter_tiles`, `iter_chunks`).
21//!
22//! # Regions and masks
23//!
24//! The [`LCRegion`] trait defines spatial subsets of a lattice. Concrete
25//! implementations include [`LCBox`], [`LCEllipsoid`], and set-algebra
26//! combinators ([`LCComplement`], [`LCIntersection`], [`LCUnion`],
27//! [`LCDifference`]). [`SubLattice`] and [`SubLatticeMut`] provide
28//! region-restricted views.
29//!
30//! # Relationship to C++ casacore
31//!
32//! The C++ casacore `Lattices` module provides `Lattice<T>`,
33//! `ArrayLattice<T>`, `PagedArray<T>`, `TempLattice<T>`, lattice
34//! iterators, and region/mask types. This Rust crate mirrors that
35//! hierarchy using idiomatic Rust traits and generics rather than
36//! C++ virtual inheritance.
37//!
38//! # Element types
39//!
40//! Only the 12 casacore-native types may be used as lattice elements,
41//! enforced by the [`LatticeElement`] trait. These correspond to the
42//! C++ template instantiations of `Lattice<T>`.
