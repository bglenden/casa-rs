SortKeyNotScalar

A sort key column contains non-scalar values.

Only scalar columns can be used as sort keys, matching C++ casacore’s runtime check in BaseTable::sort.

Fields

§column: String
§

SortKeyUnsortable

A sort key column contains values without a total order.

Complex32 and Complex64 columns have no natural total ordering and cannot be used as sort keys.

Fields

§column: String
§

SortNoKeys

No sort key columns were specified.

§

SchemaMismatch

The tables in a concatenation have incompatible schemas.

All tables passed to Table::concat must have identical schemas (same column names, types, and array shapes).

C++ equivalent: ConcatTable constructor’s schema check.

Fields

§message: String
§

ConcatEmpty

A concatenation was attempted with zero tables.

Table::concat requires at least one table.

§

ConstituentNotSaved

A constituent table in a concatenation has no disk path.

All constituent tables must be saved to disk before the ConcatTable can be persisted, since the on-disk format stores relative paths.

Fields

§index: usize
§

IndexNoColumns

No columns were supplied to crate::ColumnsIndex::new.

§

IndexColumnNotScalar

A column is non-scalar (array/record) and cannot be indexed.

Only scalar columns can serve as index keys, matching C++ casacore’s ColumnsIndex restriction.

Fields

§column: String
§

IndexColumnUnsortable

A column has an unsortable type (Complex32/Complex64) and cannot be indexed.

Complex types have no total order and therefore cannot be used as index keys.

Fields

§column: String
§

IndexNotUnique

crate::ColumnsIndex::lookup_unique found more than one matching row.

Fields

§count: usize
§

Taql(String)
