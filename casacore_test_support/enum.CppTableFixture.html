MutationRemovedColumn

Verify-only: scalar_primitives with col_str removed (any DM).

§

MutationRemovedRows

Verify-only: scalar_primitives with row 1 removed (any DM).

§

MutationAddedColumn

Verify-only: scalar_primitives + extra(Float32, 42.0) column (any DM).

§

LockFile

Lock interop: table with (id: Int, name: String), created with PermanentLocking to produce a table.lock file with sync data.

§

RefTable

RefTable interop: parent table with 3 rows (id: Int, name: String) and a RefTable selecting rows 0 and 2. The path argument is a directory containing parent.tbl/ and ref.tbl/.

§

SortedRefTable

Sorted RefTable interop: parent table with 5 rows (id: Int, name: String, value: Float), sorted descending by id and saved as a RefTable. The path argument is a directory containing parent.tbl/ and sorted.tbl/.

§

ConcatTable

ConcatTable interop: two 3-row tables (id: Int, name: String) concatenated as a ConcatTable. The path argument is a directory containing part0.tbl/, part1.tbl/, and concat.tbl/.

§

DeepCopy

Deep copy interop: a 5-row table deep-copied to a different storage manager. The path argument is a directory containing original.tbl/ and copy.tbl/.

§

ColumnsIndex

ColumnsIndex interop: table with antenna_id (Int32), 50 rows, value = row_index % 10. Used to verify ColumnsIndex lookups on C++-written data.

§

IsmScalarPrimitives

ISM scalar primitives: same schema as SsmScalarPrimitives (Bool, Int, Double, String) but stored with IncrementalStMan.

§

IsmSlowlyChanging

ISM slowly changing: table with SCAN_NUMBER (Int) and FLAG (Bool), 10 rows where values repeat across multiple consecutive rows, exercising the ISM delta-compression semantics.

§

IsmComplexScalars

ISM complex scalars: table with col_c32 (Complex) and col_c64 (DComplex), 3 rows, stored with IncrementalStMan.

§

TiledColumnStMan

TiledColumnStMan interop: Fixed-shape Float32 [2,3] array column, 3 rows, tile shape [2,3,2].

§

TiledShapeStMan

TiledShapeStMan interop: Variable-shape Float32 array column, 4 rows with two different shapes ([2,3] and [3,2]).

§

TiledCellStMan

TiledCellStMan interop: Variable-shape Float32 array column, 3 rows each with a unique shape ([2,3], [4,2], [3,3]).

§

ForwardColumn

ForwardColumnEngine interop: base table with col_value (Double, 3 rows) and a forwarding table that delegates col_value via ForwardColumnEngine. The path is the forwarding table directory; base is at {path}_base.

§

ScaledArray

ScaledArrayEngine interop: stored_col (Int array [2], 3 rows) and virtual_col (Double array, via ScaledArrayEngine with scale=2.5, offset=10.0).

§

AipsIOVariableArray

AipsIO variable-shape array: Float32 column “data” with ndim=2, 4 rows with shapes [2,3], [3,2], [3,2], [2,3], values 1.0..24.0.

§

SsmVariableArray

SSM variable-shape array: same schema and data as AipsIOVariableArray but stored with StandardStMan.

§

UndefinedScalars

Undefined scalars: 4-row table (Int, Double, String) where only rows 0 and 2 are written; rows 1 and 3 keep default values (0, 0.0, “”).

§

ColumnKeywords

Column keywords: 2-row table (flux: Double, id: Int) with table-level and per-column keywords. flux has “unit”=“Jy” and “ref_frame”=“LSRK”; id has “description”=“source identifier”.

§

AipsIORecordColumn

Record column: 3-row table (id: Int, meta: Record) with per-row record values. Row 0: {unit: “Jy”, value: 2.5}, Row 1: {flag: true}, Row 2: {} (empty).

§

MixedSchema

Mixed schema: 2-row table combining scalar (Int, Double), fixed array (Float32 [4]), variable array (Float32 2-D), record column, table keywords (telescope, version), and column keywords (flux: unit=“Jy”).

§

TableInfoMetadata

TableInfo metadata: 1-row table (id: Int) with TableInfo set to type=“Measurement”, subType=“UVFITS”. Tests table.info file interop.

§

AipsioAllNumericScalars

AipsIO all numeric scalars: 3 rows × 6 cols (uChar, Short, uShort, uInt, Float, Int64) stored with StManAipsIO.

§

AipsioComplexScalars

AipsIO complex scalars: 3 rows × 2 cols (Complex, DComplex) stored with StManAipsIO.

§

AipsioTypedArrays

AipsIO typed arrays: 3 rows × 3 cols (Int[4], Double[2,2], Float32[3]) stored with StManAipsIO.

§

Aipsio3DFixedArray

AipsIO 3D fixed array: Float32 [2,3,4], 2 rows with ascending values 1..24 and 25..48, stored with StManAipsIO.

§

SsmAllNumericScalars

SSM all numeric scalars: same as AipsioAllNumericScalars but stored with StandardStMan.

§

SsmComplexScalars

SSM complex scalars: same as AipsioComplexScalars but stored with StandardStMan.

§

SsmTypedArrays

SSM typed arrays: 3 rows × 3 cols (Int[4], Double[2,2], Complex32[2]) stored with StandardStMan.

§

AipsIOComplexVariableArray

AipsIO Complex32 variable-shape array: 4 rows with shapes [2,4], [4,2], [4,2], [2,4], values (1,0.5)..(32,16.0). Matches MS DATA column pattern.

§

SsmComplexVariableArray

SSM Complex32 variable-shape array: same data as AipsIOComplexVariableArray but stored with StandardStMan.

§

AipsioUnsignedArrays

AipsIO unsigned integer arrays: 3 rows × 3 cols (uChar[4], uShort[4], uInt[4]) stored with StManAipsIO.

§

SsmUnsignedArrays

SSM unsigned integer arrays: same as AipsioUnsignedArrays but stored with StandardStMan.

§

AipsioStringArray

AipsIO string array: 3 rows × 1 col (String[3]) stored with StManAipsIO.

§

SsmStringArray

SSM string array: same as AipsioStringArray but stored with StandardStMan.

§

AipsioComplex64Array2D

AipsIO Complex64 2D array: 3 rows × 1 col (DComplex[2,2]) stored with StManAipsIO.

§

SsmComplex64Array2D

SSM Complex64 2D array: same as AipsioComplex64Array2D but stored with StandardStMan.

§

AipsioEmptyTable

AipsIO empty table: schema with Int32 scalar + Float32[4] array, 0 rows, stored with StManAipsIO.

§

SsmEmptyTable

SSM empty table: same schema as AipsioEmptyTable but stored with StandardStMan.

§

TiledColumn3DArray

TiledColumnStMan 3D array: Float32 [2,3,4], 3 rows, tile shape [2,3,2,2].

§

IsmTypedArrays

ISM typed arrays: 3 rows × 3 cols (Int[4], Double[2,2], Bool[3]) stored with IncrementalStMan. Full CC/CR/RC cross-matrix.

§

IsmComplexArrays

ISM complex arrays: 3 rows × 2 cols (Complex32[2], Complex64[2]) stored with IncrementalStMan. Full CC/CR/RC cross-matrix.

§

IsmColumnKeywords

ISM column keywords: 2-row table (flux: Double, id: Int) with per-column keywords, stored with IncrementalStMan. Full CC/CR/RC cross-matrix.
