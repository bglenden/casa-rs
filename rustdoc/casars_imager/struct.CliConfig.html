Expand description

Parsed CLI configuration for the standalone imager.

Fields§

§ms: PathBuf

Input MeasurementSet path.

§imagename: PathBuf

Output image prefix. Products are written as PREFIX.psf, PREFIX.image, and so on.

§imsize: usize

Square image size in pixels.

§cell_arcsec: f64

Cell size in arcseconds.

§field_ids: Option<Vec<i32>>

Optional selected FIELD_IDs.

§phasecenter_field: Option<i32>

Optional FIELD_ID used as the image phase center.

§phasecenter: Option<String>

Optional explicit direction used as the image phase center.

The current frontend accepts CASA-style fixed J2000 strings such as J2000 19:59:28.500 +40.44.01.50.

§ddid: Option<i32>

Optional DATA_DESC_ID restriction.

§spw: Option<i32>

Optional spectral-window restriction when DDID is not supplied.

§spw_selector: Option<String>

Optional CASA-style SPW selector text, including channel clauses.

§channel_start: Option<usize>

Optional first selected channel.

§channel_count: Option<usize>

Optional selected-channel count.

§datacolumn: Option<String>

Optional explicit data-column override.

§save_model: SaveModelMode

CASA-style model persistence mode.

§start_model: Option<PathBuf>

Optional CASA image used to seed the initial model product.

§outlier_file: Option<PathBuf>

Optional CASA outlier-field definition file.

§correlation: Option<String>

Optional explicit scalar-plane override.

Raw-correlation overrides use XX, YY, RR, or LL. Stokes-plane overrides use I, Q, U, or V.

§spectral_mode: SpectralMode

Spectral imaging mode.

§cube_axis: CubeAxisConfig

CASA-style cube-axis configuration for specmode='cube' and specmode='cubedata'.

§weighting: WeightingMode

Visibility weighting policy.

§per_channel_weight_density: bool

CASA-style perchanweightdensity toggle for spectral cubes.

§use_pointing: bool

CASA-style usepointing toggle for POINTING-table direction corrections.

§uv_taper: Option<GaussianUvTaper>

Optional CASA-style Gaussian UV taper.

§restoring_beam_mode: RestoringBeamMode

Restoring-beam policy for restored products.

§deconvolver: Deconvolver

Requested minor-cycle deconvolver.

§nterms: usize

Requested MTMFS Taylor-term count for deconvolver='mtmfs'.

§multiscale_scales: Vec<f32>

Requested multiscale kernel sizes in pixels.

§small_scale_bias: f32

CASA-style multiscale selection bias.

§niter: usize

Minor-cycle iteration count.

§nmajor: Option<usize>

CASA-style major-cycle limit. None corresponds to nmajor=-1.

§fullsummary: bool

Include the CASA-compatible long-form summaryminor task report.

§gain: f32

Minor-cycle loop gain.

§threshold_jy: f32

Absolute CLEAN stopping threshold in Jy/beam.

§nsigma: f32

CASA-style robust-RMS stopping multiplier.

§psf_cutoff: f32

Restoring-beam fit cutoff.

§mosaic_pb_limit: f32

Mosaic primary-beam cutoff used for flat-noise normalization.

§pbcor: bool

Write CASA-style PB-corrected mosaic image products.

§write_pb: bool

Write the primary-beam product used for PB correction.

§minor_cycle_length: usize

Residual-refresh cadence.

§cyclefactor: f32

CASA-style cycle-threshold scale factor.

§min_psf_fraction: f32

Lower clamp for the PSF fraction used to derive cycle thresholds.

§max_psf_fraction: f32

Upper clamp for the PSF fraction used to derive cycle thresholds.

§hogbom_iteration_mode: HogbomIterationMode

Hogbom minor-cycle iteration accounting policy.

§use_mask: CleanMaskMode

CASA-style clean mask mode.

§auto_mask: AutoMultiThresholdConfig

CASA-style auto-multithresh controls used when use_mask is auto.

§mask_boxes: Vec<[usize; 4]>

Optional inclusive pixel-space clean boxes (x0, y0, x1, y1).

§mask_image: Option<PathBuf>

Optional CASA image mask whose non-zero pixels are cleanable.

§w_term_mode: WTermMode

Requested w-term handling mode.

§force_standard_gridder: bool

Explicit CASA-style gridder='standard' request. The MFS frontend can otherwise infer mosaic metadata from POINTING rows.

§w_project_planes: Option<usize>

Optional explicit wproject plane budget.

§dirty_only: bool

Skip CLEAN and only write dirty/residual products.

§standard_mfs_acceleration: StandardMfsAccelerationPolicy

Standard-MFS runtime acceleration policy.

§standard_mfs_backend: Option<String>

Optional explicit standard-MFS backend override.

§standard_mfs_grid_threads: Option<String>

Optional explicit standard-MFS grid worker count override.

§standard_mfs_tile_anchor: Option<String>

Optional explicit standard-MFS fixed-tile anchor override.

§standard_mfs_residual_backend: Option<String>

Optional explicit standard-MFS residual-refresh backend override.

§standard_mfs_initial_dirty_backend: Option<String>

Optional explicit standard-MFS initial dirty/PSF backend override.

§standard_mfs_metal_minor_cycle_chunk: Option<String>

Optional standard-MFS Metal minor-cycle command-buffer component chunk.

§standard_mfs_metal_grouped_input_cache: Option<bool>

Optional explicit Metal grouped input cache override.

§standard_mfs_memory_target_mb: Option<usize>

Optional standard-MFS planner memory target in MiB.

§standard_mfs_prepare_buffer_mb: Option<usize>

Optional standard-MFS prepare-buffer budget in MiB.

§imaging_memory_target_mb: Option<usize>

Optional shared imaging source-stream memory target in MiB.

§imaging_prepare_buffer_mb: Option<usize>

Optional shared imaging source-stream prepare-buffer budget in MiB.

§imaging_row_block_rows: Option<usize>

Optional shared imaging source-stream row-block override.

§imaging_prepare_workers: Option<usize>

Optional shared imaging source-stream prepare worker count.

§write_preview_pngs: bool

Write PNG preview sidecars for the CASA image products.
