Struct MeasurementSet 

Source 
pub struct MeasurementSet { /* private fields */ }
Expand description

A MeasurementSet: main table + subtables.

This is the top-level type for working with MS data. It owns the main table and all attached subtables (12 required + 5 optional).

When saved to disk, subtable links are emitted as true casacore TpTable keywords, using the same relative-path encoding that C++ MeasurementSet writes for sibling subtables.

§Example

use casacore_ms::ms::MeasurementSet;
use casacore_ms::builder::MeasurementSetBuilder;

// Create an in-memory MS
let ms = MeasurementSet::create_memory(MeasurementSetBuilder::new())
    .expect("create MS");
assert_eq!(ms.row_count(), 0);
assert_eq!(ms.subtable_ids().len(), 12);

pub fn open(path: impl AsRef<Path>) -> MsResult<Self>

Open an existing MS from disk.

Opens the main table and follows subtable keywords to open subtables. casacore-style TpTable keywords are preferred, but legacy string keywords written by older versions of this crate are also accepted.

Source

pub fn save(&mut self) -> MsResult<()>

Save the MS to its path (must have been created with a path or opened from disk).

The save step refreshes the main-table metadata and rewrites subtable keyword payloads using casacore’s relative ././SUBTABLE form.

Source

pub fn save_as(&mut self, path: impl AsRef<Path>) -> MsResult<()>

Save the MS to a new path.

All persisted subtable references are rebased so the copied MS remains self-consistent when reopened by Rust or C++ casacore tools.
